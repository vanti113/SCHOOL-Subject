工夫と鑑賞

演習24
図書管理システムの制作
課題内容
  • 一覧表示、データ追加、データ修正、データ削除とデータ検索の諸機能をまとめた「図書管理システム」に加えて、ユーザ入力の部分にJavascriptを使って入力チェックを行なうようにする
  • 表示するデータの件数表示
  • and検索のほかに、or検索も行えるようにする
  • クライアント側（ブラウザで）とサーバ側（PHPプログラムで）の両方で入力データが正当かどうかのチェック
  • HTML5の備え付けの機能を使ってデータの正当性チェック
完成ページ
  • WEB 図書管理システムHOME
ソースコード
  • 一覧表示 : main.php source
  • データ追加 : add.php source
  • データ変更 : update.php source
  • データ検索 : search.php source
  • データ削除 : delete.php source
  • テンプレート1 : paintToHtml.php source
  • テンプレート2 : paintToHtml.php source
工夫と鑑賞
  • HTML5のPattern機能の活用 : Inputタグの検証機能を利用してユーザが最初入力する時、間違っている形式やハッキング攻撃に使われるHTMLのタグを入力することを防ぐようにする。
    例：ISBNにはDBの制約に合わせた正規表現式　^([0-9]{9}[0-9xX]|[0-9]{13})$　を使って10桁の数字や9桁の数字とX又は13桁の数字だけが入力できるようにした。
    他のInputにはHTMLのタグ入力ができないよう　^[^<>]+$　をつかった。
  • SQLインジェクション攻撃対策：データベースにユーザからの入力を入れる機能を持ったファイルには別の検証メソッドを備えました。
    まずPHPのメソッド trim(), htmlentities() を利用して先頭や後ろの空白を除外、HTML文字などを無効にします。後,SQLコマンドや特殊文字を除外するために preg_replace()メソッドと正規表現式を利用して　OR, AND, WHERE, =, ; などの文字を除外します。
  • 見栄えのためのCSS,JSの活用：左のサイドバーやボタンのデザイン、全体的なイメージがよく見られるようCSSとJavascriptを利用しました。
  • 解決できなかったこと：データを修正したら本データが本来の位置ではなくどこかで移動して検索しないと見難くなることを版権しました。
    理由を考えてみましたが、どうかPSQLの整列機能と関わっているのではないかと思います。
